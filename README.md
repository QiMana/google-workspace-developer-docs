# Google Workspace Developer Docs Mirror

Repeatable markdown mirror scaffold for the Google Workspace developer corpus on `developers.google.com`.

This repo is modeled on the shape of `supabase-management-api-docs`, but the pipeline is stricter about scope measurement and uses a better markdown converter for a much larger rendered-docs surface.

## Why this repo exists

- keep the Google Workspace developer docs corpus in one auditable markdown archive
- preserve code fences, snippet languages, and DevSite examples cleanly
- make scope expansion measurable instead of hand-wavy
- separate the archive work from `qi-mono`

## Workflow

1. `bun install`
2. `bun run scrape -- --url https://developers.google.com/workspace/guides/get-started`
3. optional: `bun run measure-scope`
4. full archive pass: `bun run scrape -- --measurement measurements/generated-scope.json`

Useful flags:

- `bun run measure-scope -- --out measurements/generated-scope.json`
- `bun run scrape -- --limit 25`
- `bun run scrape -- --url https://developers.google.com/apps-script/guides/logging`
- `bun run scrape -- --overwrite`
- `bun run scrape -- --fresh`

## Repo layout

- `docs/`: generated markdown archive
- `measurements/`: machine-readable scope snapshots
- `methodology/`: human-readable explanation of counting and conversion
- `scripts/`: Bun scripts for crawling and scraping

## Markdown strategy

The conversion pipeline is intentionally closer to Obsidian Clipper than the old Supabase mirror:

- fast bulk fetch over plain HTTP for the main archive pass
- Playwright available as a rendered fallback when a page does not expose usable content over raw HTML
- `defuddle/node` for the markdown conversion layer
- Google DevSite-specific DOM cleanup before conversion so breadcrumbs, feedback chrome, and copy buttons do not leak into the archive
- nested output paths that preserve the upstream URL tree, for example:
  - `docs/workspace/guides/get-started.md`
  - `docs/apps-script/guides/logging.md`

## Notes

- `measurements/current-scope.json` is the checked-in authoritative snapshot for the verified `5,504`-page corpus boundary.
- `bun run measure-scope` generates a fresh candidate manifest at `measurements/generated-scope.json` for future reconciliation and refresh work.
- `manifest.json` is generated by `bun run scrape`.
