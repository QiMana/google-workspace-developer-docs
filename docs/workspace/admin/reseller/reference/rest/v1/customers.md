# REST Resource: customers

## Resource: Customer

When a Google customer's account is registered with a reseller, the customer's subscriptions for Google services are managed by this reseller. A customer is described by a primary domain name and a physical address.

JSON representation

```
{
  "customerId": string,
  "customerDomain": string,
  "postalAddress": {
    object (Address)
  },
  "phoneNumber": string,
  "alternateEmail": string,
  "resourceUiUrl": string,
  "customerDomainVerified": boolean,
  "kind": string,
  "customerType": enum (CustomerType),
  "primaryAdmin": {
    object (PrimaryAdmin)
  }
}
```

## Address

JSON template for address of a customer.

JSON representation

```
{
  "countryCode": string,
  "kind": string,
  "region": string,
  "addressLine2": string,
  "locality": string,
  "addressLine3": string,
  "postalCode": string,
  "addressLine1": string,
  "organizationName": string,
  "contactName": string
}
```

<table><colgroup><col width="25%"> <col></colgroup><thead><tr><th colspan="2">Fields</th></tr></thead><tbody><tr><td><code>countryCode</code></td><td><p><code>string</code></p><p>For <code>countryCode</code> information, see the <a href="http://www.iso.org/iso/country_codes.htm">ISO 3166 country code elements</a>. Verify that country is <a href="https://www.cloudconnect.goog/docs/DOC-29141">approved</a> for resale of Google products. This property is required when creating a new customer.</p></td></tr><tr><td><code>kind</code></td><td><p><code>string</code></p><p>Identifies the resource as a customer address. Value: <code>customers#address</code></p></td></tr><tr><td><code>region</code></td><td><p><code>string</code></p><p>An example of a <code>region</code> value is <code>CA</code> for the state of California.</p></td></tr><tr><td><code>addressLine2</code></td><td><p><code>string</code></p><p>Line 2 of the address.</p></td></tr><tr><td><code>locality</code></td><td><p><code>string</code></p><p>An example of a <code>locality</code> value is the city of <code>San Francisco</code>.</p></td></tr><tr><td><code>addressLine3</code></td><td><p><code>string</code></p><p>Line 3 of the address.</p></td></tr><tr><td><code>postalCode</code></td><td><p><code>string</code></p><p>A <code>postalCode</code> example is a postal zip code such as <code>94043</code>. This property is required when creating a new customer.</p></td></tr><tr><td><code>addressLine1</code></td><td><p><code>string</code></p><p>A customer's physical address. An address can be composed of one to three lines. The <code>addressline2</code> and <code>addressLine3</code> are optional.</p></td></tr><tr><td><code>organizationName</code></td><td><p><code>string</code></p><p>The company or company division name. This is required.</p></td></tr><tr><td><code>contactName</code></td><td><p><code>string</code></p><p>The customer contact's name. This is required.</p></td></tr></tbody></table>

<table><colgroup><col width="25%"> <col></colgroup><thead><tr><th colspan="2">Enums</th></tr></thead><tbody><tr><td><code>customerTypeUnspecified</code></td><td>Customer type not known</td></tr><tr><td><code>domain</code></td><td>Domained or domain-owning customers</td></tr><tr><td><code>team</code></td><td>Domainless or email-verified customers</td></tr></tbody></table>

## PrimaryAdmin

JSON template for primary admin in case of TEAM customers

JSON representation

```
{
  "primaryEmail": string
}
```

<table><colgroup><col width="25%"> <col></colgroup><thead><tr><th colspan="2">Fields</th></tr></thead><tbody><tr><td><code>primaryEmail</code></td><td><p><code>string</code></p><p>The business email of the primary administrator of the customer.</p><p>The email verification link is sent to this email address at the time of customer creation. Primary administrators have access to the customer's Admin Console, including the ability to invite and evict users and manage the administrative needs of the customer.</p></td></tr></tbody></table>
